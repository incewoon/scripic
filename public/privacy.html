Privacy Policy

Last updated: July 12, 2026

This app sends photos to Google Gemini API for AI analysis.
Analysis results are stored only on the user's device.
The app may access your device's camera roll in the background (with permission) solely to notify you when new photos accumulate.

1. Information We Collect

• Photos uploaded by the user — processed in server memory only, never stored.
• Device ID (automatically generated UUID stored on the device) – used for daily album creation limits.
• Location (when the user adds a place to an album) — GPS coordinates and place name are collected to record the location of memories.
• Perceptual Hash of review screenshots (256-bit numeric fingerprint) – stored permanently on the server to prevent duplicate SNS review submissions.
• AI-generated summaries and comments (stored locally on device).
• Background Camera Roll Access — The app periodically checks (using Android WorkManager in the background) the number of newly added photos in your device's camera roll since the last check. This is used solely to determine whether to show a local notification suggesting you create a new story. No actual photo content is read or transmitted outside the device. The count result is discarded immediately after the check.

2. Purpose of Collection

• AI photo analysis and smart album generation.
• Device ID: Managing daily creation limits (1 album per day, +1 with SNS review verification) and preventing abuse.
• Location: Allowing users to attach location information to albums they create.
• Perceptual Hash: Preventing repeated use of the same review screenshot.
• Background Photo Count Check: Detecting when 3 or more new photos have been added to the device’s camera roll while the app is closed, in order to send a local notification encouraging users to record new memories.

3. Photo Processing & Transmission

Uploaded photos are converted to base64 on the device and sent to Google Gemini API via Firebase Cloud Function (processed in-memory only).
Photos are not stored on Scripic's servers.
Gemini processes images only for the request and deletes them after analysis. Images are not used for model training under Google's paid Gemini API policy.
For SNS review screenshots: the image is processed in server memory only and is not stored. Only a 256-bit perceptual hash (numeric fingerprint) is permanently stored to prevent duplicate submissions.
Background camera roll checks: Only the count of new photos is determined locally on the device. No photo content is accessed or sent to any server.

4. Retention Period

• Device ID is retained until the user deletes the app or the limitation management purpose ends.
• Location information attached to albums is retained as long as the album exists (until the user deletes the album).
• Perceptual hashes of review images are retained permanently for duplicate prevention purposes.
• Photos and conversation data are not stored on Scripic's servers.
• Background photo count results are discarded immediately after each check and are not retained.

5. Third-Party Sharing & Processing

① Google Gemini API
Processor: Google LLC — Purpose: Image analysis and AI content generation.
Google Privacy Policy
② Firebase / Firestore (Google LLC)
Processor: Google LLC — Purpose: Storage and management of Device ID, location data attached to albums, daily usage limits, and review image perceptual hashes.
Firebase Privacy Policy
No personal information other than Device ID, location data, and perceptual hashes is shared with any third party. Background camera roll count checks are performed entirely on the device and are never transmitted.

6. Your Rights

• You can delete photos, albums, and generated content anytime within the app.
• To request deletion of server-stored data (Device ID, location data, usage records, or perceptual hashes), please contact the developer by email.

7. Contact

개인정보 처리방침

최종 수정일: 2026년 7월 12일

본 앱은 AI 분석을 위해 사진을 Google Gemini API로 전송합니다.
분석 결과는 사용자 기기에만 저장됩니다.
앱은 사용자의 동의 하에 기기 종료 상태에서도 카메라 롤에 새 사진이 쌓였는지 확인하여 알림을 보낼 수 있습니다.

1. 수집하는 개인정보

• 사용자가 업로드한 사진 — 서버 메모리에서만 처리되며 저장되지 않습니다.
• 기기 ID (기기에 저장되는 자동 생성 UUID) — 일일 앨범 생성 제한 관리에 사용됩니다.
• 위치 정보 (사용자가 앨범에 장소를 추가할 때) — 기억의 장소를 기록하기 위해 수집됩니다.
• 리뷰 스크린샷의 지각 해시 (256비트 숫자 지문) — 중복 제출 방지를 위해 서버에 영구 저장됩니다.
• AI가 생성한 요약 및 코멘트 (기기에만 저장).
• 백그라운드 카메라 롤 접근 — 앱이 종료된 상태에서도 Android WorkManager를 통해 기기의 카메라 롤에 새로 추가된 사진의 개수를 주기적으로 확인합니다. 이는 새로운 사진이 3장 이상 쌓였을 때 로컬 알림을 보내기 위한 목적으로만 사용되며, 실제 사진 내용은 읽거나 외부로 전송되지 않습니다. 확인 결과는 알림 표시 여부 결정 후 즉시 폐기됩니다.

2. 수집 및 이용 목적

• AI 사진 분석 및 스마트 앨범 생성
• 기기 ID: 일일 생성 제한 관리(하루 1개 앨범, SNS 리뷰 인증 시 +1) 및 악용 방지
• 위치 정보: 사용자가 생성한 앨범에 장소 정보를 첨부할 수 있도록 함
• 지각 해시: 동일한 리뷰 스크린샷의 반복 사용 방지
• 백그라운드 사진 개수 확인: 앱이 실행되지 않은 상태에서 기기에 새 사진이 3장 이상 추가되었는지 감지하여, 새로운 이야기를 기록할 수 있도록 로컬 알림을 제공하기 위함

3. 사진 처리 및 전송

업로드된 사진은 기기에서 base64로 변환된 후 Firebase Cloud Function을 통해 Google Gemini API로 전송됩니다(메모리에서만 처리).
사진은 Scripic 서버에 저장되지 않습니다.
Gemini는 요청에 대해서만 이미지를 처리하며, 분석 후 삭제합니다. Google의 유료 Gemini API 정책에 따라 모델 학습에 사용되지 않습니다.
SNS 리뷰 스크린샷의 경우: 이미지는 서버 메모리에서만 처리되며 저장되지 않습니다. 중복 제출 방지를 위해 256비트 지각 해시(숫자 지문)만 영구 저장됩니다.
백그라운드 카메라 롤 확인의 경우: 새 사진의 개수만 기기 내부에서 확인하며, 실제 사진 내용은 읽거나 서버로 전송되지 않습니다.

4. 보관 기간

• 기기 ID는 사용자가 앱을 삭제하거나 제한 관리 목적이 종료될 때까지 보관됩니다.
• 앨범에 첨부된 위치 정보는 앨범이 존재하는 동안 보관됩니다(사용자가 앨범을 삭제할 때까지).
• 리뷰 이미지의 지각 해시는 중복 방지 목적으로 영구 보관됩니다.
• 사진과 대화 데이터는 Scripic 서버에 저장되지 않습니다.
• 백그라운드 사진 개수 확인 결과는 매번 확인 후 즉시 폐기되며 보관되지 않습니다.

Firebase 개인정보 처리방침
