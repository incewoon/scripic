Privacy Policy

Last updated: June 21, 2026

This app sends photos to Google Gemini API for AI analysis.
Analysis results are stored only on the user's device.

1. Information We Collect

• Photos uploaded by the user — processed in server memory only, never stored.
• Device ID (automatically generated UUID stored on the device) – used for daily album creation limits.
• Perceptual Hash of review screenshots (256-bit numeric fingerprint) – stored permanently on the server to prevent duplicate SNS review submissions. The original image cannot be reconstructed from this value.
• AI-generated summaries and comments (stored locally on device).

2. Purpose of Collection

• AI photo analysis and smart album generation.
• Device ID: Managing daily creation limits (1 album per day, +1 with SNS review verification) and preventing abuse.
• Perceptual Hash: Preventing repeated use of the same review screenshot.

3. Photo Processing & Transmission

Uploaded photos are converted to base64 on the device and sent to Google Gemini API via Firebase Cloud Function (processed in-memory only).
Photos are not stored on Scripic's servers.
Gemini processes images only for the request and deletes them after analysis. Images are not used for model training under Google's paid Gemini API policy.
For SNS review screenshots: the image is processed in server memory only and is not stored. Only a 256-bit perceptual hash (numeric fingerprint) is permanently stored to prevent duplicate submissions. The original image cannot be recovered from this hash.

4. Retention Period

• Device ID is retained until the user deletes the app or the limitation management purpose ends.
• Perceptual hashes of review images are retained permanently for duplicate prevention purposes.
• Photos and conversation data are not stored on Scripic's servers.

5. Third-Party Sharing & Processing

① Google Gemini API
Processor: Google LLC — Purpose: Image analysis and AI content generation.
Google Privacy Policy
② Firebase / Firestore (Google LLC)
Processor: Google LLC — Purpose: Storage and management of Device ID, daily usage limits, and review image perceptual hashes.
Firebase Privacy Policy
No personal information other than Device ID and perceptual hashes is shared with any third party.

6. Your Rights

• You can delete photos and generated content anytime within the app.
• To request deletion of server-stored data (Device ID, usage records, or perceptual hashes), please contact the developer by email.

7. Contact

Developer: scripichelp@gmail.com

개인정보처리방침

최종 수정일: 2026년 6월 21일

본 앱은 사진 분석을 위해 Google Gemini API를 사용합니다.
분석 결과는 사용자 기기에만 저장되며, 스크립픽 서버에는 저장되지 않습니다.

1. 수집하는 개인정보 항목

• 업로드된 사진
AI 앨범 생성 및 분석을 위해 사용자가 업로드한 이미지 파일. 사진은 서버 메모리에서만 처리되며 저장되지 않습니다.
• 기기 식별자 (Device ID)
사용자의 기기에 자동 생성·저장되는 고유 식별자(UUID)를 수집합니다. 이 정보는 사용자가 직접 제공하는 것이 아니라 앱 사용 과정에서 자동으로 생성됩니다.
• 후기 이미지 수치 지문 (Perceptual Hash)
SNS 후기 인증 시 중복 방지를 위해 이미지의 64bit 수치 지문(해시값)만 서버에 영구 보관합니다. 이 값으로는 원본 이미지 복원이 불가능합니다.
• AI 분석 결과
Gemini API를 통해 생성된 대화 요약, 코멘트 등 (기기 내 저장)

2. 개인정보의 수집 및 이용 목적

• AI를 활용한 사진 분석 및 스마트 앨범 자동 생성
• 기기 식별자: 일일 앨범 생성 제한 관리 및 부정 이용 방지 (하루 1회 제한, SNS 후기 인증 시 추가 1회 제공)
• 후기 이미지 수치 지문: 동일한 후기 스크린샷의 반복 인증 방지

3. 사진의 처리 및 전송

사용자가 업로드한 사진은 다음과 같은 경로로 처리됩니다:

• 기기에서 base64 형식으로 변환된 후 Firebase Cloud Function을 경유하여 Google Gemini API로 전송됩니다.
• Scripic 서버에는 사진이 저장되지 않습니다.
• Gemini API는 요청 처리 목적으로만 이미지를 사용하며, 분석이 완료되면 해당 이미지는 Gemini 서버에서 삭제됩니다.
• Google의 Gemini Developer API(유료) 정책에 따라, 전송된 사진은 모델 학습에 사용되지 않습니다.

SNS 후기 인증 사진의 경우:
• 업로드된 이미지는 서버 메모리에서만 처리되며 저장되지 않습니다.
• 중복 인증 방지를 위해 이미지의 수치 지문(Perceptual Hash, 256bit)만 서버에 영구 보관됩니다. 이 값으로는 원본 이미지 복원이 불가능합니다.

4. 개인정보의 보유 및 이용 기간

• 기기 식별자는 사용자가 앱을 삭제하거나, 일일 제한 관리 목적이 소멸될 때까지 보유합니다.
• 후기 이미지 수치 지문(Perceptual Hash)은 중복 방지 목적이 소멸될 때까지 영구 보관됩니다.
• 사진 및 대화 내용은 Gemini API 요청 처리 후 즉시 삭제되며, Scripic 서버에는 저장되지 않습니다.
