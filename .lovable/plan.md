문제를 다시 확인한 결과, 핵심은 **스마일 이모지 자체가 아니라 1장 사진 대화에서 3번째 어시스턴트 응답이 종료 제안으로 바뀌지 않는 점**입니다.

현재 확인된 내용:
- 서버 프롬프트는 `사진 한 장당 최대 3번` 응답 후 종료 제안을 요구합니다.
- 그런데 실제 흐름에서는 3번째 응답이 `공감 멘트 + 😊`로 끝나고, `앨범으로 정리해드릴까요?` 또는 `[READY_TO_FINISH]`가 붙지 않습니다.
- 클라이언트는 종료 제안 문구나 `[READY_TO_FINISH]`가 와야 다음 종료 로직으로 넘어가므로, 이 응답이 일반 대화로 끝나면 이후 흐름이 멈춘 것처럼 보입니다.

구현 계획:
1. **종료 턴 기준을 서버 프롬프트에서 더 강하게 고정**합니다.
   - 한국어/영어 모두에서 마지막 허용 응답은 반드시 종료 제안으로 끝나게 수정합니다.
   - 감정 공감 문장 뒤에 질문을 더 이어가거나 이모지로 마무리하지 못하게 제한합니다.

2. **1장 사진 케이스를 명시적으로 보강**합니다.
   - `photoCount = 1`, `maxTurnsPerPhoto = 3`일 때 세 번째 assistant 응답은 무조건 `정리해드릴까요? + [READY_TO_FINISH]`로 유도합니다.
   - 일반적인 멀티사진 규칙과 별개로 단일 사진 흐름이 흔들리지 않도록 만듭니다.

3. **클라이언트 종료 감지 문구를 보강**합니다.
   - 종료 제안/마무리 제안 감지 정규식을 다시 점검해, 어투 변형이나 짧은 문장도 놓치지 않게 보완합니다.
   - 필요하면 assistant 응답이 질문 없이 짧은 감탄/공감문으로 끝날 때 턴 수 기준으로 종료 후보인지 함께 판단하도록 보완합니다.

4. **렌더링/후처리 때문에 문구가 사라지는지도 함께 점검**합니다.
   - 현재 UI는 `[READY_TO_FINISH]` 토큰만 제거하고 본문은 그대로 렌더링하므로, 실제로는 문구 생성 자체가 누락된 가능성이 큽니다.
   - 이 부분을 유지하되, 토큰 제거가 본문 일부를 잘못 먹지 않도록 최종 확인합니다.

5. **재현 케이스로 검증**합니다.
   - 스크린샷과 같은 1장 대화 흐름에서 3번째 assistant 응답 직후 종료 제안이 보이는지 확인
   - `😊`가 포함된 공감 멘트 이후에도 종료 제안이 이어지는지 확인
   - 이후 사용자가 `네`, `어`, `ㅇㅋ`, `마무리해줘`라고 답했을 때 바로 앨범 생성으로 이어지는지 확인

기술 메모:
- 우선 수정 대상은 `functions/src/prompts-chat.ts`와 필요 시 `src/routes/chat.tsx`입니다.
- 현재 구조상 가장 직접적인 원인은 **모델이 3번째 응답을 종료 응답으로 생성하지 않는 프롬프트 강도 부족**으로 보입니다.
- 클라이언트는 이미 종료 토큰/마무리 문구를 기준으로 후속 처리를 하므로, 서버 응답 형식을 먼저 바로잡는 것이 우선입니다.
